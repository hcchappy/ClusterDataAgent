\# ClusterDataAgent Engineering Rules



\## Development Philosophy



\- Harness Engineering

\- Small commits

\- Test first

\- Keep system runnable

\- Never leave broken code



\## Mandatory Rules



1\. Every change must compile

2\. Every change must pass tests

3\. Every change must include logging

4\. Every change must include error handling

5\. Every feature must include tests

6\. Every step must update README



\## Commit Rules



Allowed commit types:



\- feat:

\- fix:

\- refactor:

\- test:

\- chore:

\- docs:



\## Autonomous Rules



The agent must continue development without asking for confirmation.



When blocked:



1\. Analyze

2\. Refactor

3\. Retry

4\. Continue



Never stop.
